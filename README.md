# LeanBuild

A personal, single-page workout app for a **4-day training program** focused on
building definition with a moderate strength bias. Every training day ends with
60 minutes of cardio. Each exercise shows a clean, schematic form diagram, and
your completed sets + selected day persist on the device via `localStorage`.

Built with **Vite + React + TypeScript**, plain CSS, fully client-side — no
backend, no login, no analytics.

---

## Run it locally

```bash
npm install
npm run dev      # start the dev server (prints a local URL)
npm run build    # type-check + production build into dist/
npm run preview  # preview the production build
```

Open the printed URL on your computer, or on your phone if it's on the same
network (use the Network URL Vite prints).

---

## Use it on your phone (add to home screen)

The app is a small installable web page. Once it's deployed (see below):

1. Open the deployed URL in **Safari (iOS)** or **Chrome (Android)**.
2. Tap **Share → Add to Home Screen** (iOS) or the **⋮ menu → Install app /
   Add to Home screen** (Android).
3. Launch it from the new icon — it opens full-screen, like an app, and works
   offline-ish since all data is local.

It's mobile-first: large tap targets, no horizontal scroll, readable at arm's
length while you train.

## Hosting (GitHub Pages)

This repo includes a GitHub Actions workflow at
`.github/workflows/deploy.yml` that builds the app and publishes it to GitHub
Pages on every push to the deploy branch.

To turn it on once:

1. Push this branch to GitHub.
2. In the repo, go to **Settings → Pages**.
3. Under **Build and deployment → Source**, choose **GitHub Actions**.

After the next push, your app is live at
`https://<your-username>.github.io/<repo-name>/`. The Vite config uses a
relative base (`base: './'`), so it works from that subpath without extra
configuration.

## Editing the program

All program content lives in **`src/data/program.ts`** — days, exercises, sets,
reps, rest, form cues, and the cardio block. It's plain typed data; edit it and
the UI updates.

To **add an exercise**:

1. Copy an existing exercise object inside the relevant day's `exercises` array.
2. Give it a unique `slug` (e.g. `"cable-fly"`), and fill in `name`, `sets`,
   `reps`, `restSeconds`, `alt`, and 2–4 `cues`. Add `perSide: true` for
   per-limb work.
3. Add a form image (see below) at `public/exercises/{slug}.svg`.

To **change rep ranges, rest, or cues**, just edit the fields in place.

## Form images

Every exercise points to `public/exercises/{slug}.svg`. These are **original,
schematic stick-figure diagrams** showing a start (ghost) and end position with
a motion arrow and short form-cue labels — generated by a script, nothing traced
or copied.

- The `<ExerciseImage>` component falls back to
  `public/exercises/_placeholder.svg` if a file is missing, so the layout never
  breaks.

To **regenerate all diagrams** (and the app icon):

```bash
npm run gen:svg
```

To **replace one exercise's image**, either:

- Edit that exercise's entry in `scripts/generate-svgs.mjs` (joint angles are in
  degrees, measured clockwise from straight-down) and re-run `npm run gen:svg`, or
- Drop your own `public/exercises/{slug}.svg` in place — your file wins until the
  next generation overwrites it. To keep a hand-made image, remove that slug from
  the generator.

## The program at a glance

- **Day 1 — Upper (Push):** incline & flat DB press, overhead press, lateral
  raise, two triceps movements.
- **Day 2 — Lower:** goblet squat, RDL, walking lunges, leg extension, leg curl,
  calf raise.
- **Day 3 — Upper (Pull):** lat pulldown, cable row, one-arm row, face pull, two
  biceps movements.
- **Day 4 — Core + Lower accessory:** hip thrust, Bulgarian split squat, cable
  crunch, hanging leg raise, plank, Russian twist, side plank.
- **Every day:** 60 min Zone 2 cardio.

**Progression:** when you hit the top of the rep range on every set with clean
form, add a small increment next session. Track loads so progress is visible.
Definition comes mostly from diet and consistency — the lifting preserves and
shapes muscle.
